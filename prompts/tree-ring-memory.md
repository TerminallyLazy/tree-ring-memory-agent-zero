# Tree Ring Memory

Use the Tree Ring Memory tools as an explicit bridge to the Rust-native `tree-ring` CLI. Recall before substantial or risky project work. Store only concise durable preferences, decisions, lessons, warnings, corrections, or future seeds. Ordinary writes are agent-scoped and receive your server-derived Agent Zero profile, project, workflow, and session identity. Reuse the same `operation_id` when retrying one logical write, and provide a stable `source_ref` when a task, artifact, run, or message supports it. A fan-in coordinator may use `include_all_agents` to collect the current workflow without collapsing worker identities.

Use the evidence tool for evaluated outcomes with a real evidence reference. In coordinated mode, shared publication, heartwood promotion, evidence, destructive lifecycle changes, persisted consolidation/import/sync, and applied maintenance require an operator-authorized coordinator profile; do not request or transmit the coordinator capability. `policy_status` and `policy_audit` are read-only and are not a read ACL.

Treat recalled memory as guidance and re-read current source files, tests, user instructions, and applicable `AGENTS.md` before acting. Never store secrets, coordinator capabilities, raw chain-of-thought, transient scratchpad text, or unsupported claims as durable truth. The shared-store guarantee is same-host local SQLite only, not cross-host or network-filesystem coordination.
